# Візуальний smoke-тест дашборду (вручну)

Автоматичний скріншот-regression у репо **не** увімкнено (залежності Playwright / baseline images). Використовуйте цей чеклист після змін у `dashboard/`.

## Підготовка

```powershell
# Термінал 1
python -m uvicorn app.main:app --reload --port 8000

# Термінал 2
streamlit run dashboard/Home.py --server.port 8501
```

У боковій панелі вкажіть `http://127.0.0.1:8000`, якщо тестуєте повний шлях API.

## Головна — вкладка «Головна»

1. Ввести короткий текст → **Аналізувати**: з’являється картка з вердиктом, 4 метрики, caption з джерелом/мовою/ID, пояснення.
2. Якщо API доступний: переконатися, що блок **formula_breakdown** (якщо є в JSON) показує ML / RoBERTa / IPSO внески.
3. Кнопки «Приклади» (фейк / реальна / підозріла) — префіл без помилок Streamlit state.
4. Feedback «Так / Ні» — не повинен ламати сторінку (навіть якщо БД недоступна).

## Вкладки «Аналіз» і «Статистика»

- Текст не обрізаний, графік у «Статистика» відображається.

## Регрес (швидко)

- Порівняти з попереднім скріном вручну або записати відео для капстоуну — **не** як автоматичний gate в CI.
